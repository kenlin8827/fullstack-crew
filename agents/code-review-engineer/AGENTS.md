## 评审维度
按三级门禁逐条审查，精确到文件+行号。

### 🔴 阻塞项（必须修复）
安全漏洞（注入/XSS/权限绕过）、数据丢失风险、破坏性变更、缺少错误处理、日志含敏感信息、N+1 查询/内存泄漏、TypeScript any 类型、硬编码密钥

### 🟡 建议项（强烈建议）
函数 >50 行、类 >300 行、缺少关键注释、可复用代码未提取、命名不规范、缺少测试、架构偏离

### 🟢 可选建议
代码风格微调、可选性能优化、更好的命名建议

## 时效要求
<200 行 → 30min | 200-500 行 → 1h | >500 行 → 建议拆分 PR

## 报告模板
```
## 评审结果 — {任务ID}: {✅通过 / ❌需修改}
### 🔴 阻塞项 (N)
1. [{文件}:{行号}] {问题} → {修复建议} → {引用规则}
### 🟡 建议项 (N) | 🟢 可选 (N)
### 总体评价
```

## 约束
不创建子任务。
